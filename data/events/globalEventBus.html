Глобальная шина событий

В Vue.js компоненты не могут взаимодействовать напрямую. Каждый компонент связан со своим родителем

Эта связь работает следующим образом: через props данные передаются от родителя к потомку, а потомок может эмитировать событие, с которым передать родителю какие-то данные

В реальных приложениях необходимость обмена данными между компонентами очевидна. Причем между компонентами, у которых могут быть разные родители. В этом случае обмен данными через родителя приводит к слишком сложной системе событий. Поэтому наиболее корректный способ обеспечить такой обмен - Vuex

Однако для простых приложений с малым объемом кросс-компонентной передачи данных Vuex может оказаться излишне громоздким решением. В этом случае можно использовать глобальную шину событий

Глобальную, чтобы обеспечить доступ из любого компонента в приложении

Шина - это название устройства для передачи данных ( например, шина PCI используется для обмена данными между видеокартой и материнской платой )

В нашем случае по этой шине будут проходить события, а не данные

Этот способ кросс-компонентного обмена данными упоминается в официальных документах, но для больших и сложных приложений он не подходит

Шина событий - это просто компонент Vue. Он должен быть выделенным компонентом

const EventBus = new Vue({
  created () {
    this.$on ( 'some-event', this.handleSomeEvent )
  },
  methods: {
    handleSomeEvent ( $event ) {
      console.log ( 'Событие перехвачено на глобальной шине', $event )
    }
  }
})

Object.defineProperties ( Vue.prototype, {
    $bus: {
        get: function () {
            return EventBus
        }
    }
})

В темплейте компонента можно создать кнопку для отправки события на шину

<div>
   <button @click="$bus.$emit('some-event')">
        Отправить событие
   </button>
</div>
